Python Asyncio基础用法

发表于 2019-10-23 更新于 2020-05-03 分类于 编程足迹

前言

异步IO:执行操作后立即返回可以执行其他操作，任务完成后，通过状态来通知继续回来处理调用。
协程：指与调用方协作，产出由调用方提供的值的过程。由程序主动控制在单线程里执行函数A时，可随时中断去执行函数B。
事件循环：程序开启一个无限的循环，把协程对象注册到事件循环上。当满足事件发生的时候，调用相应的协程对象。所有的对象被执行完毕且不再有新协程被添加，程序就终止。

概念

早期协程基于生成器，使用yield 和 yield from 暂停将程序执行流程交给子程序实现交替执行，手动实现对IO的调度协程。

  • Future object：Future是一种特殊低层级 可等待对象，事件循环可以监视Future 对象是否完成，从而允许应用一部分等待另一部分完成一些工作，相当于结果容器。
  • 其具备如下状态：
    1、Pending
    2、Running
    3、Done
    4、Cancelled
  • Task object：Task 是 Future 的一个子类，将 coroutine 和 Future 联系在一起用来在事件循环中运行协程。若将协程封装为 Task 可给其附加上状态 。创建Task对象方式如下：
    1、asyncio.create_task()
    2、loop.create_task()
    3、asyncio.ensure_future()
  • awaitable objects：如果一个对象可以在 await语句中使用，那么它就是 可等待 对象。
  • awaitable 对象有三种：coroutines、Tasks 和 Futures，其内部必实现_await_ 方法。

await 的目的：
1、获取协程的结果
2、挂起当前协程，将控制交由事件循环，切换到其他协程，然后等待结果，最后恢复协程继续执行

启动协程

1、Python3.4 的协程是通过 @asyncio.coroutine 和 yeild from 实现。

2、Python3.5 引入 async/await 语法糖。
